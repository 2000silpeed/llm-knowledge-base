## 1. 사전 준비

### 1-1. 프로젝트 구조 확인

아래 파일/디렉토리가 있어야 합니다.

```
프로젝트 루트/
├── config/
│   ├── settings.yaml       ← LLM 설정 (모델, API 키 환경변수 등)
│   └── prompts.yaml        ← LLM 프롬프트 템플릿
├── scripts/
│   ├── query.py            ← O6 온톨로지 주입 포함
│   ├── mcp_server.py       ← O7 MCP 서버
│   └── cli.py              ← kb 명령어
├── wiki/                   ← 컴파일된 wiki 파일들 (없으면 아래 단계로 생성)
└── .kb_graph.db/           ← Kuzu 그래프 DB (없으면 아래 단계로 생성)
```

### 1-2. 환경 변수 설정

`config/settings.yaml`의 `api_key_env` 값을 확인합니다.  
기본값: `ANTHROPIC_API_KEY`

```bash
# Anthropic API 키 설정 (터미널에서 실행)
export ANTHROPIC_API_KEY="sk-ant-..."
```

키가 없으면 [console.anthropic.com](https://console.anthropic.com)에서 발급받으세요.

### 1-3. 의존성 설치

```bash
# 프로젝트 루트에서 실행
uv sync

# kuzu 설치 확인
uv run python -c "import kuzu; print('kuzu OK')"
```

### 1-4. wiki 데이터가 있는지 확인

```bash
kb status
```

출력 예시:
```
📊 지식 베이스 현황
  raw: 12개 파일
  wiki: 8개 개념 / 3개 탐색 / 0개 충돌
```

**wiki/concepts/ 가 비어 있다면** → 먼저 자료를 인제스트하고 컴파일해야 합니다:

```bash
# 예시: 웹 아티클 1개 인제스트
kb ingest https://example.com/article

# wiki 컴파일 (개념 파일 생성)
kb compile --all
```

## 2. O6 테스트 — 온톨로지 쿼리 강화

> **O6 핵심:** `kb query` 실행 시 그래프 DB와 커뮤니티 요약이 자동으로 컨텍스트에 추가됩니다.

### 단계 1. 그래프 DB 초기화 (처음 한 번만)

```bash
# 그래프 DB 스키마 생성
kb graph init
```

성공 출력 예:
```
✓ 그래프 DB 초기화 완료
  DB 경로: .kb_graph.db
  테이블: Concept, Domain, ActionType + 8개 관계 타입
```

### 단계 2. 온톨로지 추출 (wiki → triple)

```bash
# wiki/concepts/ 의 모든 개념 파일에서 관계 triple 추출
kb ontology extract --all
```

추출이 완료되면 `.kb_concepts/` 폴더에 `*.triples.json` 파일이 생깁니다.

```bash
# 생성된 triple 파일 확인
ls .kb_concepts/
```

### 단계 3. 그래프 적재 (triple → Kuzu DB)

```bash
kb graph load
```

성공 출력 예:
```
✓ 그래프 적재 완료
  적재된 개념: 8개 / triple: 42개
```

### 단계 4. 커뮤니티 분석 (선택 — 더 풍부한 컨텍스트)

```bash
# CO_OCCURS 클러스터를 분석해 wiki/_communities.json 생성
kb graph analyze --communities
```

생성 확인:
```bash
cat wiki/_communities.json | python3 -m json.tool | head -30
```

출력 예:
```json
{
  "generated_at": "2026-04-19",
  "total_communities": 2,
  "communities": [
    {
      "id": "고객세분화",
      "size": 4,
      "members": ["고객세분화", "RFM분석", "코호트분석", "CLV"],
      "summary": "고객 행동 분석과 세분화를 위한 방법론 클러스터..."
    }
  ]
}
```

### 단계 5. 온톨로지 주입 활성화 확인

```bash
# settings.yaml에서 query_inject 확인
grep -A 4 "^ontology:" config/settings.yaml
```

출력:
```yaml
ontology:
  auto_load: false
  query_inject: true   ← 이 값이 true 이어야 함
  budget_ratio: 0.15
  max_concepts: 3
```

### 단계 6. 온톨로지 주입 포함 쿼리 실행

```bash
# 기본 쿼리 (온톨로지 자동 주입)
kb query "고객세분화란 무엇인가?"
```

출력에서 확인할 포인트:
```
[답변 내용...]

컨텍스트: 3,420 토큰 / 189,000 토큰 (5개 파일)  /  온톨로지: 커뮤니티 1개 / 개념 확장 2개
```

`온톨로지: 커뮤니티 N개 / 개념 확장 N개` 부분이 나오면 O6가 정상 동작하는 것입니다.

### 단계 7. 온톨로지 상세 확인 (--verbose)

```bash
kb query "고객세분화란 무엇인가?" --verbose
```

추가 출력:
```
온톨로지 컨텍스트 (O6):
  커뮤니티: ['고객세분화']
  개념 확장: ['고객세분화', 'RFM분석']
  토큰: 487
```

### 단계 8. 온톨로지 없이 쿼리 비교

```bash
# 온톨로지 주입 없이 실행
kb query "고객세분화란 무엇인가?" --no-ontology
```

`온톨로지:` 줄이 사라지고 토큰 수가 줄어들면 정상입니다.

## 3. O7 테스트 — MCP 서버

> **O7 핵심:** Claude Desktop 같은 MCP 호스트에서 이 지식 베이스를 도구로 사용할 수 있습니다.

### 3-1. MCP 서버 단독 실행 테스트

터미널에서 직접 JSON-RPC 메시지를 보내 테스트합니다.

#### 방법 A: 파이프로 테스트 (간단)

```bash
# 서버 초기화 메시지 전송
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' \
  | uv run python -m scripts.mcp_server
```

기대 출력 (한 줄):
```json
{"jsonrpc": "2.0", "id": 1, "result": {"protocolVersion": "2024-11-05", "capabilities": {"tools": {"listChanged": false}}, "serverInfo": {"name": "kb-knowledge-base", "version": "1.0.0"}}}
```

#### 방법 B: 도구 목록 조회

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}
{"jsonrpc":"2.0","method":"notifications/initialized"}
{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}' \
  | uv run python -m scripts.mcp_server
```

기대 출력 (2줄 — initialize 응답 + tools/list 응답):
```
{"jsonrpc": "2.0", "id": 1, "result": {"protocolVersion": ...}}
{"jsonrpc": "2.0", "id": 2, "result": {"tools": [{"name": "search_concepts", ...}, ...]}}
```

`"tools"` 배열에 6개 항목이 나오면 정상입니다.

#### 방법 C: search_concepts 도구 호출

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}
{"jsonrpc":"2.0","method":"notifications/initialized"}
{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"search_concepts","arguments":{"query":"고객","limit":5}}}' \
  | uv run python -m scripts.mcp_server
```

기대 출력 (id 2의 결과):
```json
{"jsonrpc": "2.0", "id": 2, "result": {"content": [{"type": "text", "text": "검색: '고객' — 3건\n- [concepts/고객세분화] 고객세분화\n..."}], "isError": false}}
```

#### 방법 D: get_hierarchy 도구 호출

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}
{"jsonrpc":"2.0","method":"notifications/initialized"}
{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"get_hierarchy","arguments":{"concept":"고객세분화"}}}' \
  | uv run python -m scripts.mcp_server
```

#### 방법 E: query_knowledge 도구 호출 (LLM 호출 발생 — API 비용 주의)

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}
{"jsonrpc":"2.0","method":"notifications/initialized"}
{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"query_knowledge","arguments":{"question":"RFM 분석이란?"}}}' \
  | uv run python -m scripts.mcp_server
```

### 3-2. Claude Desktop 연동 설정

#### 설정 스니펫 출력

```bash
# Claude Desktop용 설정 출력
kb mcp config --host claude-desktop
```

출력 예:
```
설정 파일 위치: ~/Library/Application Support/Claude/claude_desktop_config.json

추가할 내용:
{
  "mcpServers": {
    "kb": {
      "command": "/usr/bin/python3",
      "args": ["-m", "scripts.mcp_server"],
      "cwd": "/home/user/project"
    }
  }
}
```

#### Claude Desktop에 등록하는 방법

1. 위 명령어(`kb mcp config --host claude-desktop`)로 스니펫을 확인합니다.
2. 아래 경로의 파일을 텍스트 에디터로 엽니다:
   - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
   - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
3. 파일이 없으면 새로 만들고, 있으면 `mcpServers` 키에 `kb` 항목을 추가합니다.
4. Claude Desktop을 **완전히 종료** 후 재시작합니다.
5. Claude Desktop 채팅창에서 망치 아이콘(🔨)을 클릭하면 `kb` 도구 목록이 보입니다.

#### VS Code 연동 (선택)

```bash
kb mcp config --host vscode
```

출력된 설정을 `.vscode/mcp.json`에 저장하면 VS Code Copilot이 kb 도구를 사용할 수 있습니다.

## 4. 문제 해결 (FAQ)

### Q: `kb graph init` 실행 시 "kuzu를 찾을 수 없습니다" 오류

```bash
uv add kuzu
uv sync
```

### Q: `kb ontology extract --all` 실행 시 wiki/concepts/ 가 비어 있다고 나옴

`kb compile --all`을 먼저 실행해 wiki 개념 파일을 생성해야 합니다.

### Q: `kb query` 에서 `온톨로지:` 줄이 나오지 않음

가능한 원인:
1. **그래프 DB 없음** → `kb graph init` + `kb graph load` 실행
2. **쿼리 키워드와 개념명이 매칭 안 됨** → 더 구체적인 개념명으로 질문
3. **settings.yaml에서 비활성화** → `ontology.query_inject: true` 확인

확인 방법:
```bash
kb query "질문" --verbose 2>&1 | grep -i "O6\|온톨로지\|communities"
```

### Q: MCP 서버 테스트 시 아무 출력도 없음

MCP 서버 로그는 stderr로 출력됩니다. 아래처럼 stderr를 분리해서 확인하세요:

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' \
  | uv run python -m scripts.mcp_server 2>/tmp/mcp_err.log

# stdout (JSON-RPC 응답)
cat /tmp/mcp_out.log

# stderr (로그)
cat /tmp/mcp_err.log
```

### Q: `get_hierarchy` / `get_causal_chain` 결과가 "(관계 없음)"

그래프 DB에 triple이 없는 경우입니다. `kb ontology extract --all` → `kb graph load` 순서로 실행했는지 확인하세요:

```bash
kb graph stats
```

출력 예:
```
Concept 노드: 8개
IS_A 엣지: 3개
PART_OF 엣지: 2개
...
```

모두 0이면 `kb graph load` 를 다시 실행합니다.

### Q: `query_knowledge` 도구 호출 시 API 오류

`ANTHROPIC_API_KEY` 환경 변수가 MCP 서버 프로세스에도 전달되어야 합니다. Claude Desktop `mcp_servers.json`에 `env` 키를 추가하세요:

```json
{
  "mcpServers": {
    "kb": {
      "command": "python3",
      "args": ["-m", "scripts.mcp_server"],
      "cwd": "/path/to/project",
      "env": {
        "ANTHROPIC_API_KEY": "sk-ant-..."
      }
    }
  }
}
```

## 전체 테스트 체크리스트

아래를 순서대로 체크하면 O6~O7이 정상 동작하는 것을 확인할 수 있습니다.

```
[ ] 1. uv sync 완료 + kuzu import OK
[ ] 2. ANTHROPIC_API_KEY 설정
[ ] 3. wiki/concepts/ 에 파일 1개 이상 존재
[ ] 4. kb graph init 성공
[ ] 5. kb ontology extract --all 성공 (.kb_concepts/*.triples.json 생성)
[ ] 6. kb graph load 성공 (Concept 노드 1개 이상)
[ ] 7. kb graph analyze --communities 성공 (wiki/_communities.json 생성) [선택]
[ ] 8. kb query "질문" → 온톨로지: 커뮤니티 N개 / 개념 확장 N개 출력 확인 (O6)
[ ] 9. echo '...' | uv run python -m scripts.mcp_server → JSON 응답 확인 (O7)
[ ] 10. kb mcp config --host claude-desktop → 설정 스니펫 출력 확인 (O7)
```
