# CLAUDE.md — LLM 지식 베이스 프로젝트 작업 규약

## 프로젝트 정체성

- **프로젝트명:** LLM 기반 개인 지식 베이스 시스템
- **채널:** @소담 AI 스튜디오 / @sodam_ai
- **참조 원전:** Karpathy 트윗 (2039805659525644595)
- **핵심 철학:** LLM이 위키를 쓰고 유지한다. 사람은 자료 투입과 질문만.

## 핵심 문서 위치

```
LLM_지식베이스_기획서.md   — 전체 설계 원본 (변경 시 반드시 동기화)
task.md                    — 작업 단위 목록 및 진행 상태 (작업 기준 문서)
handoff.md                 — 완료 작업 핸드오프 기록 (세션 간 연속성)
```

## Claude 작업 규칙

### 작업 시작 시
1. `task.md`를 읽고 현재 상태 파악
2. `handoff.md`의 마지막 항목을 읽고 직전 세션 맥락 확인
3. 작업할 태스크 ID를 명시하고 시작

### 작업 중
- 태스크 단위로 작업하고, 완료 즉시 `task.md` 상태 업데이트
- 설계 결정이 바뀌면 `기획서.md`도 함께 수정
- 파일 생성/수정 후 반드시 `cokacdir --sendfile`로 사용자에게 전송

### 작업 완료 시
- `task.md`에서 해당 태스크를 `[x]`로 표시
- `handoff.md`에 핸드오프 항목 추가 (아래 형식 준수)
- 다음 태스크 제안

### 핸드오프 항목 형식
```
## HO-{번호} | {날짜} | {태스크 ID}
**완료:** {무엇을 만들었는지 — 파일명 포함}
**결정사항:** {설계 결정, 트레이드오프 선택 이유}
**주의:** {다음 작업자가 반드시 알아야 할 것}
**다음:** {이어서 해야 할 태스크 ID}
```

## 코드 규칙

- 언어: Python 3.11+
- 패키지 관리: `pyproject.toml` (uv 사용)
- 파일 인코딩: UTF-8
- 마크다운: CommonMark 준수
- `raw/` 디렉토리는 절대 수정하지 않음 — 읽기 전용
- 모든 LLM 호출은 `config/settings.yaml`의 모델 설정을 따름
- 청킹 로직은 `context_limit` 기반 자동 계산 — 하드코딩 금지

## 설계 원칙 (변경 불가)

1. **raw/ 불변** — 원본 자료는 항상 원형 보존
2. **wiki/ LLM 소유** — 위키 파일은 LLM이 쓰고 유지, 사람이 직접 편집 최소화
3. **모델 독립** — settings.yaml 한 줄 변경으로 모든 모델 지원
4. **복리 우선** — 모든 탐색 결과는 wiki/로 환원
5. **RAG 없음** — 벡터 DB/임베딩 사용 금지, 인덱스 파일로 해결
