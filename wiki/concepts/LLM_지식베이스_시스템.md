# LLM 지식베이스 시스템

LLM이 위키를 직접 쓰고 유지하는 개인 지식 관리 시스템. 사람은 자료 투입과 질문만 담당하고, LLM이 구조화·요약·연결을 책임진다.

## 핵심 파이프라인

1. **인제스트** — URL·PDF·Office 파일 → `raw/` 마크다운 변환
2. **컴파일** — `raw/` 마크다운 → LLM → `wiki/concepts/` 항목 생성
3. **질의** — 우선순위 기반 컨텍스트 조립 → LLM 답변
4. **복리** — 탐색 결과를 `wiki/explorations/`에 저장 → 다음 질의의 컨텍스트로 재사용

## 설계 원칙

- **raw/ 불변**: 원본 자료는 항상 원형 보존, 절대 수정 금지
- **RAG 없음**: 벡터 DB·임베딩 대신 마크다운 인덱스 파일로 처리 (Karpathy 검증 방식)
- **모델 독립**: `config/settings.yaml`의 `context_limit` 한 줄 변경으로 전체 파이프라인 자동 적응
- **청킹 자동화**: 문서 크기에 따라 단일패스 / Map-Reduce / 계층트리 자동 선택

## 청킹 전략

| 문서 크기 | 전략 | 방식 |
|---|---|---|
| 가용 토큰의 ≤80% | 단일 패스 | 전체를 한 번에 처리 |
| ≤300% | Map-Reduce | 청크별 병렬 요약 → 통합 |
| 300% 초과 | 계층 트리 | 2단계 요약 트리 |

## CLI 사용법

```bash
kb ingest <파일/URL>          # 자료 투입
kb compile [--all|--changed]  # 위키 컴파일
kb query "질문 내용"           # 질의
kb status                     # 현황 요약
kb watch                      # 파일 감시 + 자동 컴파일
```
